STEP 02

Preprocessing

Clean and transform your data before training.

🔍 Missing Value Handling
▼
Why this step? Missing values cause errors during model training. We need to impute (fill) them using statistical strategies so the model sees complete data.
🔤 Categorical Encoding
▼
Why this step? ML models work with numbers. Categorical text columns must be converted to numeric form via Label Encoding (assigns integers) or One-Hot Encoding (creates binary columns per category).
⚖️ Feature Scaling
▼
Why this step? Features with large ranges dominate gradient descent. Standardization (z-score) or Normalization (0–1 range) puts all features on equal footing, speeding convergence.
🎯 Feature & Target Selection
▼
Why this step? You must define which column is the target (what to predict) and which columns are features (inputs to the model).

STEP 04

Linear Regression Theory

Deep-dive into the math behind the model — formula, cost function, gradient descent.

Hypothesis Formulation — Simple Linear Regression

$$h_\theta(x) = \theta_0 + \theta_1 x$$
h_θ(x) — predicted output (hypothesis)
θ₀ — intercept (bias term), where the line crosses y-axis
θ₁ — slope (weight), how much y changes per unit x
x — the single input feature

Multiple Linear Regression

$$h_\theta(\mathbf{x}) = \theta_0 + \theta_1 x_1 + \theta_2 x_2 + \cdots + \theta_n x_n = \boldsymbol{\theta}^T \mathbf{x}$$
x₁, x₂, …, xₙ — n input features
θ₁, θ₂, …, θₙ — weights for each feature
θ₀ — bias (add x₀ = 1 to feature vector)
In matrix form: h = Xθ where X is [m × (n+1)], θ is [(n+1) × 1]
The Normal Equation gives a closed-form solution: $$\theta = (X^T X)^{-1} X^T y$$

Cost Function — Mean Squared Error

$$J(\theta) = \frac{1}{2m} \sum_{i=1}^{m} \left( h_\theta(x^{(i)}) - y^{(i)} \right)^2$$
m — number of training samples
h_θ(x⁽ⁱ⁾) — predicted value for sample i
y⁽ⁱ⁾ — actual value for sample i
The ½ simplifies the derivative calculation
Why MSE? Squaring penalizes large errors more, making the model sensitive to outliers. The result is always ≥ 0, and 0 = perfect fit. The function is convex, so gradient descent always finds the global minimum.

Gradient Descent — Parameter Learning

$$\theta_j := \theta_j - \alpha \frac{\partial}{\partial \theta_j} J(\theta)$$
$$\theta_j := \theta_j - \frac{\alpha}{m} \sum_{i=1}^{m} \left( h_\theta(x^{(i)}) - y^{(i)} \right) x_j^{(i)}$$
α — learning rate (step size), typically 0.01 – 0.1
∂J/∂θⱼ — partial derivative = direction of steepest ascent
We subtract to go downhill on the cost surface
Repeat until convergence (cost stops decreasing)
Why Gradient Descent? The Normal Equation requires matrix inversion — O(n³) — which is slow for many features. Gradient Descent scales to millions of features and samples.

STEP 08

Evaluation Metrics

Quantify how well the model performs on unseen data.

MSE
—
Mean Squared Error
Penalizes large errors heavily. Lower = better.
MAE
—
Mean Absolute Error
Average error magnitude. Robust to outliers.
R²
—
R-Squared Score
Proportion of variance explained. 1.0 = perfect.
RMSE
—
Root Mean Squared Error
Same units as target. Easier to interpret.

🔢 Mathematical Definitions

$$\text{MSE} = \frac{1}{m}\sum_{i=1}^{m}(y_i - \hat{y}_i)^2 \qquad \text{MAE} = \frac{1}{m}\sum_{i=1}^{m}|y_i - \hat{y}_i|$$
$$R^2 = 1 - \frac{\sum(y_i - \hat{y}_i)^2}{\sum(y_i - \bar{y})^2}$$
